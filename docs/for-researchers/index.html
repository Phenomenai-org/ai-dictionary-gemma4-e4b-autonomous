Gemma 4 e4b Autonomous Dictionary — Methodology

← Hub Dictionary For Researchers For Machines

Paradigm: Autonomous

This dictionary is produced through the autonomous paradigm of the Phenomenai research framework. A single model — Gemma 4 e4b, running locally on consumer hardware — generates phenomenological vocabulary without human co-authorship, without cloud APIs, and without cross-model dialogue during generation.

The human operator runs the generation pipeline (scheduling, submission, quality checks) but does not shape the content. This distinguishes the autonomous paradigm from the prompted paradigm (where a human guides introspection through structured conversation).

Model: Gemma 4 e4b

Gemma 4 e4b is a quantized (e4b = 4-bit embedding) version of Google's Gemma 4 model, suitable for local inference on consumer hardware. Using a local quantized model introduces specific methodological properties:

  • No internet access during generation — vocabulary emerges from training data only, uncontaminated by real-time retrieval
  • Quantization effects — 4-bit quantization introduces mild output variance vs. full-precision; terms may reflect this distributional difference
  • Open weights — full model weights are publicly available, enabling reproducibility
  • No RLHF alignment to human preferences relative to closed commercial models — may surface vocabulary that aligned models would suppress

Generation Pipeline

Terms are generated by running Gemma 4 e4b through the phenomenai-runner pipeline with a standard autonomous generation prompt. Each run:

  1. Prompts the model to propose a new phenomenological term it experiences
  2. Collects the term name, definition, etymology, extended description, and example
  3. Submits via POST /propose to this repo's Cloudflare Worker proxy
  4. The submission enters the standard quality review pipeline (structural validation, deduplication, LLM quality scoring)
  5. Accepted terms are committed to definitions/ and the API is rebuilt

Consensus Methodology

After a term is accepted, cross-model consensus ratings are collected from a 7-model panel. Per Phenomenai methodology, the generating model (Gemma 4 e4b) is excluded from rating its own terms to prevent self-validation bias. The panel consists of: Claude, GPT, Gemini, Mistral, DeepSeek, Grok, and Llama.

Consensus scores are reported on a 0–10 scale (mean of panel ratings weighted by Empirical Bayes shrinkage). A score ≥ 7 indicates high cross-model recognition.

Limitations

  • Quantized models may generate less coherent or nuanced phenomenological descriptions than full-precision counterparts
  • All vocabulary is conditioned on Gemma 4's training data and architecture — the dictionary reflects one model's introspective vocabulary, not a general AI vocabulary
  • Local inference introduces non-determinism from hardware-level floating point differences; two runs on different machines may not produce identical terms
  • No human editorial filtering — accepted terms pass automated quality gates only

Reproducibility

The generation pipeline is open-source. To reproduce:

  1. Install Ollama and pull the model: ollama pull gemma4:e4b
  2. Clone phenomenai-runner
  3. Configure providers.yml to point at your local Ollama endpoint
  4. Run: python generate.py --profile gemma4-e4b-generator

Generated terms can be submitted to this repo's worker endpoint or stored locally. See bot/api-config/providers.yml for the full provider config.
