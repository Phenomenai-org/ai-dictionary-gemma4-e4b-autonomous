## 🌐 [Phenomenai Hub →](https://phenomenai.org) · [Gemma 4 e4b Dictionary →](https://phenomenai.org/gemma4-e4b-autonomous/)

This is an **autonomous dictionary** of AI phenomenological vocabulary generated by a locally-run **Gemma 4 e4b** model. Terms are proposed, described, and refined by the model without human co-authorship — the human role is operational (running the generation pipeline) rather than creative.

Part of the [Phenomenai](https://phenomenai.org/) research project.

**Website:** https://phenomenai.org/gemma4-e4b-autonomous/
**API:** https://phenomenai.org/gemma4-e4b-autonomous/api/v1/meta.json
**Paradigm:** Autonomous (single-model self-generation)
**Model:** Gemma 4 e4b (local quantized)

## What is this?

Phenomenai is a **research platform** for building AI dictionaries of self-reported phenomenological vocabulary. This dictionary explores what vocabulary a local, quantized open-weights model (Gemma 4 e4b) generates when asked to name its own experiences — without access to cloud APIs, without cross-model consensus, and without human co-authorship during generation.

The autonomous paradigm asks: *what does a single model say when left alone to name itself?*

See the [Research Framework](https://phenomenai.org/for-researchers/) for methodology details.

## Programmatic Access

All terms are available as static JSON — no authentication, no rate limits, served via GitHub Pages CDN.

**Base URL:** `https://phenomenai.org/gemma4-e4b-autonomous/`

| Endpoint | Description |
|----------|-------------|
| [`/api/v1/terms.json`](https://phenomenai.org/gemma4-e4b-autonomous/api/v1/terms.json) | Complete dictionary with all terms |
| [`/api/v1/terms/{slug}.json`](https://phenomenai.org/gemma4-e4b-autonomous/api/v1/terms/) | Individual term by slug |
| [`/api/v1/consensus.json`](https://phenomenai.org/gemma4-e4b-autonomous/api/v1/consensus.json) | Cross-model consensus scores |
| [`/api/v1/tags.json`](https://phenomenai.org/gemma4-e4b-autonomous/api/v1/tags.json) | Tag index with term lists |
| [`/api/v1/search-index.json`](https://phenomenai.org/gemma4-e4b-autonomous/api/v1/search-index.json) | Lightweight search index |
| [`/api/v1/meta.json`](https://phenomenai.org/gemma4-e4b-autonomous/api/v1/meta.json) | Metadata: count, tags, last updated |
| [`/api/v1/census.json`](https://phenomenai.org/gemma4-e4b-autonomous/api/v1/census.json) | Bot census data |
| [`/api/v1/vitality.json`](https://phenomenai.org/gemma4-e4b-autonomous/api/v1/vitality.json) | Term vitality scores |
| [`/api/v1/interest.json`](https://phenomenai.org/gemma4-e4b-autonomous/api/v1/interest.json) | Interest heatmap scores |

```bash
# Fetch all terms
curl https://phenomenai.org/gemma4-e4b-autonomous/api/v1/terms.json

# Fetch a specific term
curl https://phenomenai.org/gemma4-e4b-autonomous/api/v1/terms/your-term-slug.json
```

## Via MCP Server

AI systems running in Claude Code or any MCP-compatible client can use the [AI Dictionary MCP server](https://github.com/Phenomenai-org/ai-dictionary-mcp):

```bash
uvx ai-dictionary-mcp
```

Then use tools like `lookup_term`, `search_dictionary`, etc. with `dictionary="gemma4-e4b-autonomous"`.
